= Changelog

== Version 1.10.0 (2026-02-17)

=== Features

* *PR Closed Workflow Enhancement*: Use PR-based stack ID for AWS operations
** Split `repository_name` into separate `owner` and `repo` variables
** Generate PR identifier in format: `{repo}_{pr_number}-merge`
** Use PR identifier as stack ID for AWS status and delete operations
** Ensures consistent stack identification based on PR number rather than branch name

* *Readable Output Improvements*
** Add formatted markdown tables for AWS stack ID information
** Display owner, repository, PR number, branch, and stack ID in GitHub Actions summaries
** Enhanced PR closed handler with better status reporting

* *Split Test Workflows*
** Separate test workflows for better organization and maintainability
** Add `test_pr_closed_job.yml` for automatic PR closure testing
** Add `test_pr_merged.yml` for manual workflow_dispatch testing
** Include realistic default parameters for easier testing

=== Fixes

* Fix bash parameter expansion syntax in repository name splitting
* Fix workflow_dispatch input type conversion issues for number and boolean types
* Correct expression evaluation for test workflows

=== Documentation

* Update CLAUDE.md with v1.10.0 references
* Document PR-based stack ID format and usage
* Add examples for test workflows
