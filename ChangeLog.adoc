= Changelog

== Version 1.17.0 (2026-03-05)

=== Features

* *`reusable_run_it.yml`*: Add `playwright_screenshots_path` input parameter
** Configurable path for Playwright screenshots upload (default: `IT/target/playwright-screenshots/**`)
** Fixes typo in previous hardcoded path (`playright` → `playwright`)

* *`reusable_run_it.yml`*: Add `$GITHUB_STEP_SUMMARY` output to key steps
** `set env variables`: markdown table with `AWS_PUBLIC_DNS_HOSTNAME`, `IT_FOLDER_EXISTS`, `DATAGEN_FOLDER_EXISTS`, `BONITA_URL`, `playwright_screenshots_path`
** `upload Playwright screenshots`: summary with artifact name and path
** `upload Bonita test reports`: summary with artifact name and path
** Artifact names include `${{ github.run_id }}` suffix; `retention-days: 30` set on both uploads

* *`test_run_it.yml`*: Rewrite to use mocked artifacts and proper inputs
** `workflow_dispatch` inputs: `bonita_url` and `playwright_screenshots_path` with realistic defaults
** New `mock_artifacts` job creates and uploads `aws_instance.yaml` and `pre_requisites.yaml` locally
** Eliminates full chain (`prerequisites → create_server → build_sca → deploy_sca`) from test workflow

== Version 1.16.0 (2026-03-05)

== Version 1.14.0 (2026-02-26)

=== Features

* *List AWS Servers Workflow*
** Added `reusable_list_server.yml` workflow using `-c list` command
** Added `test_list_server.yml` test workflow
** Displays instances as a formatted markdown table in step summary
** Columns: Name, Status, DNS (public + private), Launch Time
** Sorted by status (running first), then launch time (newest first)

* *Replace Maven with GitHub Release Download*
** Replaced `./mvnw dependency:copy` with `gh release download` from `laurentleseigneur/bonita-aws`
** Applies to `reusable_create_server.yml`, `reusable_status_server.yml`, `reusable_pr_closed.yml`
** No longer requires Maven, `pom.xml`, or JFrog credentials to fetch the jar
** Uses `GHP_TOKEN` for access to the private repository

=== Fixes

* Use `GHP_TOKEN` instead of `github.token` for `gh release download` (private repo access)
* Fix bonita-aws release tag format to `bonita-aws-X.X`

== Version 1.12.0 (2026-02-17)

=== Features

* *UIB AWS Deployment Health Check*
** Added automatic health check that waits for UIB container to be ready before deploying
** Polls `http://localhost/uib/` endpoint every 5 seconds
** Accepts HTTP status codes: 200 (OK), 302 (Redirect), or 401 (Unauthorized)
** 5-minute timeout (300 seconds) with retry mechanism
** Eliminates 502 Bad Gateway errors that previously required manual job replay

* *Test Workflow Improvements*
** Enhanced test job configurations with additional parameters
** Removed unnecessary jobs for better workflow efficiency

=== Documentation

* Updated CLAUDE.md with UIB health check documentation
* Updated all version references to v1.11.0 and v1.12.0
* Documented health check behavior, timeout, and accepted status codes
* Added health check section to UIB deployment workflows documentation

== Version 1.11.0 (2026-02-17)

=== Features

* *PR-based Stack ID for AWS Operations*
** Split `repository_name` into separate `owner` and `repo` variables
** Generate PR identifier in format: `{repo}_{pr_number}-merge`
** Use PR identifier as stack ID for AWS status and delete operations
** Ensures consistent stack identification based on PR number rather than branch name
** Add `pr_identifier` as job output for downstream consumption

* Fix workflow_dispatch input type conversion issues for number and boolean types
* Simplify test workflow expressions to use `github.event.inputs` with `fromJSON()`
* Add realistic default parameters to test_pr_merged.yml

=== Documentation

* Update CLAUDE.md with PR-based stack ID documentation
* Document new variables and outputs
* Add examples for test workflows
